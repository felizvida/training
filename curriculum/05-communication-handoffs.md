# Module 5: Communication, Handoffs, and Lessons Learned

## Purpose

Teach participants to turn completed Codex work into durable notes that another staff member can resume.

## Good Close-Outs Include

- What was requested.
- What files, data, and commands were used.
- What changed.
- What was verified.
- What remains.
- Any privacy or source limitations.
- Exact paths to important artifacts.

## Handoff Prompt

```text
Document this thoroughly so a new Codex thread or another staff member can continue.
Include the goal, current state, key files, commands already run, verification status, known limitations, and next recommended steps.
Do not include secrets, raw private data, or unnecessary identifiers.
```

## Lessons-Learned Prompt

```text
Write a concise GitHub-safe lessons-learned source memo.
Keep it short and structured.
Include context, what went wrong or was discovered, durable lesson, recommended control or workflow change, and open questions.
Do not include passwords, secrets, raw data, private notes, hostnames, or ticket exports.
```

## Status Report Prompt

```text
Inspect the actual workflow, output files, or verification logs.
Give me a status report with: current state, evidence checked, completed items, blockers, and next action.
Do not infer success from surrounding context.
```

## Instructor Notes

Scientific staff often do work that needs to be resumed weeks later. A good Codex close-out is less about elegance and more about preserving the actual retrieval handles: paths, commands, dates, scripts, outputs, and caveats.
