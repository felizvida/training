# Exercise 5: Task Triage

## Scenario

Scientific staff often know their work pain points, but not which ones are good Codex tasks. This exercise turns real work into safe, concrete Codex prompts.

## Goal

Identify three tasks Codex could assist with and write one high-quality prompt.

## Instructions

Choose three tasks from your own work or from the examples below.

Example tasks:

- Summarize a scientific software inventory.
- Compare a new export with a prior snapshot.
- Refresh a local dataset using a documented workflow.
- Check whether a scheduled workflow finished successfully.
- Turn operational notes into a sanitized lessons-learned memo.
- Build a restart handoff for a project.
- Review a code or documentation change for risk.
- Create synthetic data and an answer key for a training session.

For each task, fill in:

| Task | Input | Output | Privacy boundary | Verification |
| --- | --- | --- | --- | --- |
|  |  |  |  |  |

Then choose one task and write a prompt with:

```text
Goal:
Source:
Audience:
Privacy boundary:
Output:
Verification:
What not to assume:
```

## Debrief

- Which task is safest to try first?
- Which task would save repeated effort?
- Which task needs the strongest privacy boundary?
- Which task needs live or current verification?
- Which task should become a documented workflow?
