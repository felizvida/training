Portfolio section: Unit 7

## Goal

Create a reusable handoff or source memo, understand this training repo as a Codex-assisted work product, and learn how memory or skill patterns can be shared safely.

## Anchored Experience

This unit models durable restart notes, GitHub-safe source memos, sanitized memory/skill patterns, release notes, and the training repository's own maintenance as a legitimate Codex task.

## Why This Matters

Good work should survive the end of a chat, a meeting, or a week away. This unit turns completed work into something another person, or a future Codex thread, can pick up without archaeology.

## Starter Prompt

```text
Create a sanitized restart handoff for the work completed so far in this self-paced tutorial.
Include goal, current state, key files, commands or checks, outputs, verification status, limitations, and next steps.
Do not include secrets, raw private data, names, emails, hostnames, or row-level identifiers.
```

## Deliverable

One of:

- Restart handoff.
- Source memo.
- Sanitized memory/skill pattern.
- Training repo stewardship note.

## Finish Line

You are done when your note lets someone continue without asking you to reconstruct the whole story. The satisfying result is a clean handoff that makes future work easier.

## Self-Check

- Could another person continue from the note?
- Are paths and checks specific?
- Are privacy boundaries explicit?
- Are open issues separated from completed work?
