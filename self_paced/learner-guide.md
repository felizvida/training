# Learner Guide

## Tutorial Promise

By the end of this tutorial, you should be able to turn a real scientific-support task into a safe, source-grounded Codex request with an output, verification plan, and handoff.

This tutorial is not about clever prompts. It is about giving Codex the right evidence, boundaries, and definition of done.

## Working Rhythm

Each unit is designed for 60 minutes:

| Time | Activity |
| --- | --- |
| 0-10 min | Read the unit context and learning goals |
| 10-20 min | Inspect the source material |
| 20-40 min | Draft or run the Codex prompt |
| 40-50 min | Review output against the checklist |
| 50-60 min | Save the unit deliverable and reflection |

## Before Unit 1

Open these three files:

- [README.md](README.md)
- [portfolio-template.md](portfolio-template.md)
- [progress-tracker.md](progress-tracker.md)

The portfolio is where your final work goes. The progress tracker is your quick view of what remains.

## If You Have Codex Access

Use the starter prompts directly, but ask Codex to inspect files before answering. Keep each final response concise and evidence-grounded.

## If You Do Not Have Codex Access

You can still complete the tutorial. For each unit:

1. Write the prompt you would give Codex.
2. Write the output shape you expect.
3. Complete the self-check manually.
4. Save the deliverable in [portfolio-template.md](portfolio-template.md).

## Ground Rules

- Use the synthetic data in this repository.
- Do not add real names, emails, device IDs, private notes, ticket exports, or secrets.
- Treat the "other staff/researcher ideas" as optional context only.
- Keep the anchored project patterns at the center of the work.
- Verify before claiming success.

## If You Adapt This To Real Work

Use a sanitized description first. Do not paste private data, real identifiers, exports from restricted systems, or unpublished scientific details unless you have permission and a clear audience boundary.

## Self-Paced Support Pattern

When stuck, ask:

- What local evidence should Codex inspect?
- What should the output look like?
- What should be excluded?
- What privacy boundary applies?
- What would prove the answer is good enough?
- What should a future person need to continue?
