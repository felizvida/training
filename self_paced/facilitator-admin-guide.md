# Facilitator And Admin Guide

This guide supports an asynchronous or lightly facilitated cohort using the 8-unit self-paced tutorial.

## Recommended Delivery

| Week | Units | Suggested touchpoint |
| --- | --- | --- |
| 1 | Units 1-2 | Kickoff note and office hour |
| 2 | Units 3-4 | Privacy and source-boundary check |
| 3 | Units 5-6 | Verification and uncertainty review |
| 4 | Units 7-8 | Portfolio review and completion |

For a faster cohort, learners can complete two units per day across one week.

## What To Send Learners

Send:

- [README.md](README.md)
- [learner-guide.md](learner-guide.md)
- [portfolio-template.md](portfolio-template.md)
- [completion-checklist.md](completion-checklist.md)

Tell learners to keep outputs synthetic or sanitized.

## Review Method

Use [../assessment/rubric.md](../assessment/rubric.md) plus these self-paced criteria:

- The learner can name the source evidence.
- The learner can state what Codex should not assume.
- The learner keeps output privacy appropriate to the audience.
- The learner verifies with a check, command, source audit, or explicit manual review.
- The learner writes a handoff that another person could continue.

## Suggested Feedback Language

- "Name the source file and columns more explicitly."
- "This needs a privacy boundary before it is shareable."
- "The answer sounds plausible, but what verified it?"
- "Keep the unresolved case unresolved rather than forcing a match."
- "This is a good output, but it needs a restart note."

## Completion Artifact

Ask learners to submit one completed [portfolio-template.md](portfolio-template.md). It should not contain sensitive source data.
